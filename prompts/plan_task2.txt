Bước 1: Thu thập và phân tích thông tin học sinh
Mục tiêu

Thu thập các thông tin cần thiết để hệ thống hiểu:

Học sinh đang học gì.
Muốn kiểm tra phần nào.
Mục đích tạo đề là gì.
Mức điểm mong muốn.
Kiến thức hiện tại của học sinh ra sao.
Có yêu cầu đặc biệt nào không.

Sau đó chuẩn hóa thành một hồ sơ đầu vào thống nhất để các bước tiếp theo (xây dựng ma trận đề, sinh đề, kiểm tra chất lượng đề...) sử dụng.

Nguyên tắc thiết kế
1. Học sinh không phải giáo viên

Không yêu cầu học sinh cung cấp:

Ma trận đề.
Bloom Taxonomy.
Chuẩn đầu ra.
Tỷ lệ câu dễ/trung bình/khó.
Rubric chấm điểm.

Các thông tin này sẽ được hệ thống tự suy luận ở các bước sau.

2. Thông tin cá nhân hóa là tùy chọn

Nếu học sinh chưa từng sử dụng hệ thống:

Không có hồ sơ học tập
→ Hệ thống vẫn tạo được đề

Nếu học sinh đã có lịch sử học tập:

Có hồ sơ học tập
→ Hệ thống tạo đề cá nhân hóa hơn
3. Học sinh cung cấp dữ liệu thực tế

Thay vì hỏi:

Bạn mạnh phần nào?
Bạn yếu phần nào?

Hệ thống thu thập:

Đạo hàm: 8 điểm
Giới hạn: 5 điểm

Sau đó tự suy luận:

Đạo hàm → khá

Giới hạn → cần củng cố thêm
Các nhóm thông tin cần thu thập
Nhóm 1: Thông tin bắt buộc
Môn học

Ví dụ:

{
  "môn_học": "Toán"
}

Các giá trị có thể có:

Toán
Vật lý
Hóa học
Sinh học
Ngữ văn
Tiếng Anh
Lịch sử
Địa lý
Tin học
...
Khối lớp

Ví dụ:

{
  "khối_lớp": 11
}
Phạm vi kiểm tra

Ví dụ:

{
  "phạm_vi_kiểm_tra": "Chương Đạo hàm"
}

Hoặc:

{
  "phạm_vi_kiểm_tra": [
    "Giới hạn",
    "Đạo hàm"
  ]
}
Nhóm 2: Thông tin khuyến khích
Mục đích

Ví dụ:

{
  "mục_đích": "Luyện thi"
}

Các giá trị:

Ôn tập
Kiểm tra nhanh
Luyện thi
Đánh giá năng lực
Loại đề

Ví dụ:

{
  "loại_đề": "Trắc nghiệm"
}

Các giá trị:

Trắc nghiệm
Tự luận
Kết hợp
Số lượng câu hỏi

Ví dụ:

{
  "số_câu_hỏi": 20
}
Thời gian làm bài

Ví dụ:

{
  "thời_gian_làm_bài": 45
}

Đơn vị:

Phút
Nhóm 3: Thông tin cá nhân hóa
Mục tiêu điểm số

Ví dụ:

{
  "mục_tiêu_điểm_số": 8
}

Ý nghĩa:

5 điểm → thiên về cơ bản

7 điểm → cơ bản + vận dụng

8 điểm → đề cân bằng

9+ điểm → tăng câu vận dụng cao
Hồ sơ kiến thức hiện tại

Ví dụ:

{
  "hồ_sơ_kiến_thức": [
    {
      "chủ_đề": "Đạo hàm",
      "điểm": 8
    },
    {
      "chủ_đề": "Giới hạn",
      "điểm": 5
    }
  ]
}

Hoặc:

{
  "hồ_sơ_kiến_thức": [
    {
      "chủ_đề": "Đạo hàm",
      "mức_độ": "Cơ bản"
    },
    {
      "chủ_đề": "Giới hạn",
      "mức_độ": "Khá"
    }
  ]
}

Hệ thống sẽ tự chuyển đổi thành đánh giá nội bộ.

Ghi chú

Ví dụ:

{
  "ghi_chú": "Tập trung phần cực trị"
}

Các ví dụ khác:

Bỏ qua phần đạo hàm lượng giác

Tạo đề giống thi cuối kỳ

Tăng số câu vận dụng

Chỉ ôn phần đã học trên lớp
Kết quả đầu ra của Bước 1

Sau khi thu thập và chuẩn hóa, hệ thống tạo một hồ sơ học sinh thống nhất.

Ví dụ:

{
  "môn_học": "Toán",
  "khối_lớp": 11,
  "phạm_vi_kiểm_tra": [
    "Đạo hàm"
  ],

  "mục_đích": "Luyện thi",
  "loại_đề": "Trắc nghiệm",
  "số_câu_hỏi": 20,
  "thời_gian_làm_bài": 45,

  "mục_tiêu_điểm_số": 8,

  "hồ_sơ_kiến_thức": [
    {
      "chủ_đề": "Đạo hàm",
      "điểm": 8
    },
    {
      "chủ_đề": "Giới hạn",
      "điểm": 5
    }
  ],

  "ghi_chú": "Tập trung phần cực trị"
}
